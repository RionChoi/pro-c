# Session 06: 포인터 & 참조 (Pointers & References)

## 학습 목표
- C++에서 포인터와 참조자의 차이를 명확히 이해한다.
- `new`/`delete`를 활용한 동적 메모리 관리법을 복습한다.
- 포인터와 참조자를 함수 인자로 넘기는 시기를 구분한다.

## 핵심 개념

### 포인터 vs 참조자
```cpp
int value = 42;

// 포인터 — 주소값을 저장, nullptr 가능, 다시 가리킬 대상 변경 가능
int* ptr = &value;
*ptr = 10;          // value가 10으로 변경

// 참조자 — 반드시 초기화해야 하며 별칭 역할, 다른 대상 가리킬 수 없음
int& ref = value;
ref = 20;           // value가 20으로 변경
```

### 주요 차이점
| 항목 | 포인터 | 참조자 |
|------|--------|--------|
| 초기화 | 선택 (nullptr 가능) | 필수 |
|再할당 | 가능 | 불가능 |
| 문법 | `*ptr`, `&var` | `ref` (일반 변수처럼 사용) |
| 배열/함수 인자 | 주로 사용 | 주로 사용 |

### 동적 메모리 (C++ 스타일)
```cpp
int* p = new int(5);   // 할당
delete p;              // 해제

int* arr = new int[n](); // 배열 동적 할당
delete[] arr;
```
