# 세션 08: 생성자와 소멸자 (Constructors & Destructors)

## 학습 목표

- 클래스의 인스턴스화 시 자동 호출되는 생성자(Constructor)를 이해한다.
- 객체 수명 종료 시 호출되는 소멸자(Destructor)의 역할을 이해한다.
- 생성자 오버로딩과 초기화 리스트(Initialization List)를 익힌다.
- 깊은 복사(Deep Copy)와 얕은 복사(Shallow Copy)의 차이를 이해한다.

---

## 핵심 개념

### 1. 생성자 (Constructor)
- 객체가 생성될 때 자동으로 호출되는 특수 멤버 함수.
- 클래스 이름과 동일하며 반환 타입이 없다.
- 주로 멤버 변수의 초기화와 자원 할당을 담당한다.

```cpp
class MyClass {
public:
    MyClass() { /* 기본 생성자 */ }
    MyClass(int val) : data(val) { /* 초기화 리스트를 사용한 생성자 */ }
private:
    int data;
};
```

### 2. 소멸자 (Destructor)
- 객체가 범위를 벗어나거나 `delete`될 때 자동으로 호출된다.
- `~클래스이름()` 형식을 가지며 반환 타입과 매개변수가 없다.
- 할당된 메모리 해제, 파일 닫기 등 자원 정리를 담당한다.

### 3. 초기화 리스트 (Member Initializer List)
- 생성자 몸체(`{}`)가 실행되기 전에 멤버 변수를 초기화한다.
- `const` 멤버나 참조 멤버는 반드시 초기화 리스트에서 초기화해야 한다.

### 4. 복사 생성자 (Copy Constructor)
- 객체를 다른 객체로 초기화할 때 호출된다.
- 기본적으로는 멤버별 얕은 복사가 수행된다.
- 포인터 멤버가 있는 경우 깊은 복사를 직접 구현해야 한다.

## 과제 설명

### homework.cpp — 클래스 초기화 실습
- `Student` 클래스를 설계한다. (이름, 나이, 점수 멤버 포함)
- 기본 생성자, 매개변수 생성자, 소멸자를 구현하여 호출 시점을 로그로 출력한다.
- 초기화 리스트를 사용하여 멤버 변수를 초기화한다.

### homework2.cpp — 깊은 복사와 얕은 복사
- 동적 할당된 문자열을 멤버로 가진 `String` 클래스를 작성한다.
- 기본 생성자와 소멸자에서 메모리 할당/해제를 수행한다.
- 복사 생성자를 구현하여 깊은 복사(Deep Copy)를 수행하고, 구현하지 않았을 때 발생하는 문제(더블 프리 등)를 이해한다.

### game1.cpp — 펫 키우기 게임 (Digital Pet)
- `Pet` 클래스를 생성하여 생성자에서 이름을 짓고 소멸자에서 작별 인사를 한다.
- 생성자에서 초기 상태(배고픔, 행복도)를 설정한다.
- 다양한 활동을 통해 객체의 상태를 변경하고, 프로그램 종료 시(객체 소멸 시) 최종 상태를 보고한다.
