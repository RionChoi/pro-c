# Session 36: 직렬화 & 이진 프로토콜 (Serialization & Binary Protocol)

## 학습 목표
- 직렬화(Serialization) 및 역직렬화 기초
- 이진 프로토콜 설계 및 구현
- 메모리 버퍼 관리 (std::vector, std::stringstream)
- 엔디언(Endianness) 처리
- 커스텀 이진 형식 정의

## 핵심 개념

### 1. 직렬화의 필요성
- 메모리 데이터 → 파일/네트워크로 저장/전송
- 플랫폼 간 데이터 교환
- 상태 저장 및 복원

### 2. 텍스트 vs 바이너리
- 텍스트: 가독성 높음, 크기 큼 (JSON, XML)
- 바이너리: 크기 작음, 처리 빠름, 해석 어려움

### 3. 메모리 관리
- std::vector를 바이트 버퍼로 사용
- 읽기/쓰기 포인터 관리
- 크기 제한 체크

### 4. 엔디언
- Big-Endian: 네트워크, 상위 바이트 우선
- Little-Endian: x86/x64 CPU 기본
- 플랫폼 간 호환성 고려 필요
