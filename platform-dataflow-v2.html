Platform Data Flow & API Architecture

Request → Auth → API → Kafka → DB (Multi-Tenant)

Request Flow

  • • User → Next.js App (HTTPS)
  • • Middleware: JWT verify + Tenant routing
  • • API Route: Zod validation → Prisma
  • • Response: JSON + Set-Cookie

Auth Flow

  • • Login → Google OAuth 2.0
  • • Callback → Account + Session created
  • • JWT issued → HttpOnly Cookie
  • • NextAuth v5 · @repo/auth shared

Multi-Tenant DB

  • • Schema-per-tenant (search_path)
  • • Railway managed PostgreSQL
  • • @repo/db singleton via Prisma
  • • Tenant isolation: strongest

Kafka Events

  • • Domain events → publishEvent()
  • • Confluent Cloud (prod)
  • • docker-compose Kafka (local)
  • • KafkaJS in web-main/lib

CI/CD Pipeline

  • • GitHub Actions: lint → type-check → build
  • • Vercel: auto-deploy on main
  • • Railway: DB migrations on deploy
  • • Prisma migrate: never on prod directly

Shared Packages

  • • @repo/db — Prisma singleton
  • • @repo/auth — NextAuth factory
  • • @repo/shared-types — Zod + events
  • • @repo/ui — shadcn/ui components
