# 세션 15: 재귀 (Recursion)

## 학습 목표
- 재귀 함수의 개념과 구조를 이해한다.
- 기본 재귀 알고리즘을 직접 구현한다.
- 재귀 vs 반복의 차이와 언제 재귀를 사용해야 하는지 판단한다.
- 재귀적 사고방식(분할 정복)을 습득한다.

## 핵심 개념

### 재귀란?
자기 자신을 호출하는 함수.一个问题을 같은 구조의 더 작은 부분문제로分解하여 해결한다.

### 재귀의 구성 요소
1. **Base Case (기준 조건)**: 재귀를 멈추는 조건 — 없으면 무한 루프
2. **Recursive Case (재귀 조건)**: 자기 자신을 더 작은 매개변수로 호출

### 재귀 함수 예시
```c
int factorial(int n) {
    if (n <= 1) return 1;          // base case
    return n * factorial(n - 1);   // recursive case
}
```

### 재귀 vs 반복
| 기준 | 재귀 | 반복 |
|------|------|------|
| 코드 가독성 | 높음 | 중간 |
| 메모리 사용 | 스택 사용 (추가) | 변수만 사용 |
| 속도 | 상대적으로 느림 | 상대적으로 빠름 |
| 적합한 경우 | 트리, 분할 정복, 백트래킹 | 단순 반복 연산 |

## 세션 과제

### 과제 1: 팩토리얼 & 피보나치 (homework.c)
- `factorial(n)`: 재귀로 팩토리얼 계산
- `fibonacci(n)`: 재귀로 피보나치 수 계산
- `fibonacci_iter(n)`: 반복 버전과 비교
- 사용자로부터 양의 정수 N을 입력받아 계산

### 과제 2: 문자열 뒤집기 & 배열 합산 (homework2.c)
- `reverse_string(s, left, right)`: 재귀적으로 문자열 뒤집기
- `sum_array(arr, n)`: 재귀적으로 배열의 합 계산
- `gcd(a, b)`: 유클리드 호제법으로 최대공약수 계산 (재귀 버전)
- `is_palindrome(s, left, right)`: 펠린드롬 판별

### 게임 1: 하노이의 탑 (game1.c)
- 원반 N개를 시작rod에서 목적rod로 최소 이동回数로 옮기기
- 중간rod를 보조로 사용
- 매 이동마다 현재 상태 시각화
- 원반 수가 1~8개까지 선택 가능
