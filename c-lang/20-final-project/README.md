# 세션 20: C 최종 프로젝트 (Final Project)

## 학습 목표
- C 언어全程에서 습득한 모든 개념을综合하여 완전한 프로그램을 만든다.
- 파일 I/O, 동적 메모리, 자료구조(연결 리스트, 해시 테이블), 에러 처리를 모두 활용한다.
- 메뉴 기반 인터페이스와 데이터 persistence(영속성)를 구현한다.
- 모듈 설계를 통해 코드 구성력(self-contained项目)을 보인다.

## 프로젝트 개요: 개인 주소록管理系统 (Contact Manager)

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 연락처 추가 | 이름, 전화번호, 이메일을 입력해 저장 |
| 연락처 검색 | 이름 또는 전화번호로 검색 |
| 연락처 삭제 | 목록에서 선택하여 삭제 |
| 전체 목록 | 등록된 모든 연락처를 페이지 네비게이션으로 출력 |
| 파일 저장/불러오기 | 연락처를 CSV 파일로 저장하고 불러오기 |
| 중복 확인 | 해시 테이블로 전화번호 중복을 사전 차단 |

### 자료구조
- **연결 리스트**: 연락처의 동적 저장 (삽입, 삭제, 순회)
- **해시 테이블**: 전화번호 기반 중복 확인 및 빠른 검색
- **파일 I/O**: CSV 포맷으로 저장/불러오기

### 시스템 구성
```
main()          — 메뉴 루프, 사용자 입력 처리
ContactList     — 연결 리스트 기반 연락처 목록 관리
Contact         — 연락처 하나의 데이터 구조체
PhoneIndex      — 해시 테이블 기반 전화번호 인덱스
file_ops.c/h    — CSV 저장/불러오기
```

## 과제 구성
| 파일 | 내용 |
|------|------|
| `homework.c` | 주소록 핵심 기능 4가지 (추가, 검색, 삭제, 목록) — 연결 리스트 사용 |
| `homework2.c` | 전화번호 중복 체크 + CSV 파일 저장/불러오기 + 해시 테이블 인덱스 |
| `game1.c` | 최종 완성版 — 모든 기능을 갖춘 대화형 주소록管理系统 (메뉴, 검색, 페이지 네비게이션) |

## 연습 문제
1. 이름순 자동 정렬 기능을 추가한다.
2. 그룹(가족, 친구, 회사)별로 필터링한다.
3. 수정(업데이트) 기능을 추가한다.
4. 최대 저장 가능한 연락처 수 제한을 설정하고 확장 시 재동적 할당을 수행한다.
